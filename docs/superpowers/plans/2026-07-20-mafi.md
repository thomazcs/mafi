# MAFI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** PWA estático mobile-first de gestão de pacientes, agenda semanal e pagamentos para fisioterapeuta (spec: `docs/superpowers/specs/2026-07-20-mafi-design.md`).

**Architecture:** SPA Vite+React+TS sem backend. Store central (context+reducer) persistido em localStorage com `schemaVersion`. Lógica de negócio pura em `src/logic/*` (testada com Vitest); UI em `src/ui/*` (validação manual do Thom). Navegação por 4 abas via estado local (sem router).

**Tech Stack:** Vite 6, React 18, TypeScript, Vitest. Zero outras dependências de runtime.

## Global Constraints

- Idioma da UI: português (BR). Datas exibidas `dd/mm`; armazenadas ISO `YYYY-MM-DD`.
- Sem backend, sem router, sem lib de estado externa, sem CSS framework.
- Testes: **só** em `src/logic/` e `src/store/reducer.ts`. Rodar apenas o arquivo afetado (`npx vitest run <arquivo>`); suite completa só no fechamento.
- Commits pequenos por task, mensagens em pt-BR, **sem Co-Authored-By**.
- Visual: fundo `#FAFAF8`, texto `#1C1B1A`, cinzas neutros, accent verde-água `#0D9488`, radius 12px, fonte system-ui, touch targets ≥44px.
- Dias da semana: `0=dom … 6=sáb`. Dinheiro: number em reais, exibido `R$ 120` (sem centavos se inteiro).

```ts
// Todas as datas são strings ISO 'YYYY-MM-DD', tratadas como data local (sem TZ).
function parse(iso: string): Date { const [y, m, d] = iso.split('-').map(Number); return new Date(y, m - 1, d) }
function toISO(d: Date): string {
  const p = (n: number) => String(n).padStart(2, '0')
  return `${d.getFullYear()}-${p(d.getMonth() + 1)}-${p(d.getDate())}`
}
export function todayISO(): string { return toISO(new Date()) }
export function weekdayOf(iso: string): number { return parse(iso).getDay() }
export function monthOf(iso: string): string { return iso.slice(0, 7) }
export function addDays(iso: string, n: number): string { const d = parse(iso); d.setDate(d.getDate() + n); return toISO(d) }
export function daysUntilMonthEnd(iso: string): number {
  const d = parse(iso)
  const last = new Date(d.getFullYear(), d.getMonth() + 1, 0).getDate()
  return last - d.getDate()
}
export function weekDates(iso: string): string[] {
  const start = addDays(iso, -weekdayOf(iso))
  return Array.from({ length: 7 }, (_, i) => addDays(start, i))
}
export function formatBR(iso: string): string { return `${iso.slice(8, 10)}/${iso.slice(5, 7)}` }
export function formatMoney(v: number): string {
  return Number.isInteger(v) ? `R$ ${v}` : `R$ ${v.toFixed(2).replace('.', ',')}`
}
const MESES = ['janeiro','fevereiro','março','abril','maio','junho','julho','agosto','setembro','outubro','novembro','dezembro']
export function monthLabel(month: string): string { return MESES[Number(month.slice(5, 7)) - 1] }
```

```ts
export type Action =
  | { type: 'ADD_PATIENT'; patient: Patient }                    // se pacote=dez_sessoes: cria PackageCycle(inicio=dataInicio) + Payment(ciclo=cycleId, data=dataInicio)
  | { type: 'UPDATE_PATIENT'; patient: Patient }                 // troca de pacote p/ dez_sessoes sem ciclo aberto → cria ciclo+payment como no ADD
  | { type: 'REMOVE_PATIENT'; id: string }                       // remove patient + exceptions + attendances + payments + cycles dele
  | { type: 'ARCHIVE_PATIENT'; id: string; arquivado: boolean }
  | { type: 'ADD_EXCEPTION'; exception: ExceptionEntry }
  | { type: 'REMOVE_EXCEPTION'; id: string }
  | { type: 'CHECK_ATTENDANCE'; patientId: string; data: string }   // cria Attendance; dez: openCycle.sessoesUsadas+1; avulso: cria Payment(ciclo=data, data, valor)
  | { type: 'UNCHECK_ATTENDANCE'; patientId: string; data: string } // desfaz tudo do CHECK
  | { type: 'RENEW_PACKAGE'; patientId: string; data: string }      // novo PackageCycle(inicio=data, sessoesUsadas=0) + Payment(ciclo=novo id, data)
  | { type: 'ENSURE_MONTHLY_PAYMENTS'; month: string }              // p/ cada mensal ativo com dataInicio <= fim do mês: cria Payment(ciclo=month, data=1º dia do mês) se não existir
  | { type: 'SET_PAID'; paymentId: string; pago: boolean; data: string } // pago=true seta dataPagamento=data; false remove
  | { type: 'SET_TEMPLATES'; templates: Settings['templates'] }
  | { type: 'IMPORT_STATE'; state: AppState }
```

Casos (um `it` cada, todos assertando sobre o estado retornado):
1. `ADD_PATIENT` mensal: adiciona em `patients`, não cria cycle nem payment.
2. `ADD_PATIENT` dez_sessoes: cria 1 `PackageCycle` (`inicio` = dataInicio, `sessoesUsadas` 0) e 1 `Payment` não pago com `ciclo` = id do ciclo e `valor` = valor do paciente.
3. `REMOVE_PATIENT` limpa patients, exceptions, attendances, payments, cycles do paciente (montar estado com um registro de cada antes).
4. `CHECK_ATTENDANCE` (paciente dez_sessoes já adicionado): cria attendance e incrementa `sessoesUsadas` do ciclo.
5. `CHECK_ATTENDANCE` avulso: cria attendance + payment não pago com `ciclo` = data e `valor` = valor do paciente.
6. `CHECK_ATTENDANCE` duplicado (mesmo patientId+data): não duplica attendance nem incrementa de novo.
7. `UNCHECK_ATTENDANCE` dez: remove attendance e decrementa; avulso: remove o payment daquela data.
8. `RENEW_PACKAGE`: paciente dez com ciclo `sessoesUsadas` 9 → novo ciclo zerado + novo payment; ciclo antigo permanece.
9. `ENSURE_MONTHLY_PAYMENTS '2026-07'`: cria payment `ciclo='2026-07'` pra mensal ativo; segunda chamada não duplica; não cria pra arquivado nem pra `dataInicio` futura (`2026-09-01`); não cria pra avulso/dez.
10. `SET_PAID` true seta `pago` e `dataPagamento`; false limpa ambos.
11. `UPDATE_PATIENT` mudando pacote avulso→dez_sessoes cria ciclo+payment; mensal→mensal não cria nada.
12. `IMPORT_STATE` substitui o estado inteiro.
